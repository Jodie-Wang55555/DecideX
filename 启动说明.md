# DecideX 完整系统启动说明

## 系统架构

```
前端 (端口 8000) → 后端代理 (端口 8123) → LangGraph Dev Studio (端口 8123)
```

## 启动步骤

### 方式一：使用 LangGraph Dev Studio（推荐）

1. **启动 LangGraph Dev Studio**（这是您的后端服务）：
```bash
uvx --refresh \
    --from "langgraph-cli[inmem]" \
    --with-editable . \
    langgraph dev
```

2. **启动前端**：
```bash
cd frontend
python3 -m http.server 8000
```

3. **访问前端界面**：
   - 打开浏览器访问：http://localhost:8000
   - 前端会自动连接到 LangGraph Dev Studio (http://localhost:8123)

### 方式二：使用后端代理服务

如果您想使用自定义的后端代理（用于调试或扩展功能）：

1. **安装依赖**（如果还没有）：
```bash
pip install fastapi uvicorn python-multipart
```

2. **启动后端代理**：
```bash
./start_backend.sh
# 或直接运行
python3 backend_proxy.py
```

3. **启动前端**：
```bash
cd frontend
python3 -m http.server 8000
```

4. **访问前端界面**：
   - http://localhost:8000

## 可用的智能体

根据 `langgraph.json` 配置，系统包含以下智能体：

- **supervisor** - 监督者模式（可用于客户支持、医生预约）
- **swarm** - 群体模式（可用于预订服务）
- **decision-agent** - 决策智能体（综合成本、风险、价值分析）

## 前端智能体映射

前端界面中的智能体按钮会映射到对应的 LangGraph graph：

- 客户支持 → `supervisor`
- 医生预约 → `supervisor`
- 预订服务 → `swarm`

## 故障排除

### 前端无法连接后端

1. **检查后端是否运行**：
   - LangGraph Dev Studio 应该显示在 http://localhost:8123
   - 或者后端代理服务应该显示在 http://localhost:8123

2. **检查端口冲突**：
   - 确保 8000 端口（前端）和 8123 端口（后端）没有被占用

3. **检查 CORS 设置**：
   - 后端代理已配置 CORS，允许前端访问

### LangGraph Dev Studio 无法启动

1. **检查依赖**：
```bash
pip install -e .[inmem] langgraph-cli
```

2. **检查环境变量**：
   - 确保 `.env` 文件存在并包含必要的 API 密钥

3. **检查配置文件**：
   - 确保 `langgraph.json` 配置正确

## 开发建议

1. **同时运行两个终端**：
   - 终端1：运行 LangGraph Dev Studio
   - 终端2：运行前端服务器

2. **查看日志**：
   - LangGraph Dev Studio 会显示详细的执行日志
   - 后端代理服务会显示 API 请求日志

3. **API 文档**：
   - 如果使用后端代理，可以访问 http://localhost:8123/docs 查看 API 文档
